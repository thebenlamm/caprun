# Requirements: AgentOS

**Defined:** 2026-06-29
**Core Value:** A kernel-confined worker's only egress is broker-mediated plan
nodes, and a genuinely-propagated taint chain deterministically blocks
value-injection at sensitive sink arguments.

Requirement IDs are preserved verbatim from the canonical build order in
`planning-docs/PLAN.md` (via ingest synthesis) so they remain traceable to the
SPEC's "Done when" clauses.

## v1 Requirements

Requirements for v0 (the initial release). Each maps to exactly one roadmap
phase. **v0 DONE = REQ-s9-acceptance-test passing.** Substrate alone is not
sufficient.

### Substrate (M0)

- [x] **REQ-runtime-core**: `runtime-core` crate — core domain types with no
      I/O. *Done when:* Intent, Session, Event, Artifact, and the 3-class Effect
      enums compile.

- [x] **REQ-api-stub-plan-node**: Broker `submit_plan_node()` API surface, shape
      locked from day one. *Done when:* `submit_plan_node()` exists and returns
      `NotImplemented`; the PlanNode/ValueNode shape is locked.

- [x] **REQ-sandbox**: `sandbox` crate — kernel-enforced confinement boundary.
      *Done when:* a worker starts with CPU, memory, broker UDS, and zero
      ambient fs/net/shell; negative assertions hold (cannot read `~/.ssh`,
      cannot reach network, cannot exec un-allowlisted binaries).

- [x] **REQ-brokerd-core**: `brokerd` core — control plane / reference monitor.
      *Done when:* Session create, SQLite audit-DAG append, and UDS IPC all work.

- [x] **REQ-adapters-fs**: `adapters/fs` — filesystem adapter via fd-pass.
      *Done when:* the broker opens a workspace file and passes the fd via
      SCM_RIGHTS.

- [x] **REQ-substrate-demo**: End-to-end substrate demonstration proving complete
      mediation (no LLM required). *Done when:* `caprun` confined worker reads a
      file via passed fd and the event appears in the audit DAG.

### Design Gate (M0-design)

Hard gate: `crates/executor` must NOT be written until both docs are reviewed.
Runs in **parallel** with Substrate (M0).

- [x] **REQ-design-taint-model**: `DESIGN-taint-model.md` design doc. *Done
      when:* it explicitly states the dynamic-taint default, the hard
      planner/worker split for Tier 3+, and the I0 intent/session-creation rule
      (draft-only when seeded from untrusted content).

- [x] **REQ-design-plan-executor**: `DESIGN-plan-executor.md` design doc. *Done
      when:* it specifies ValueNode, PlanNode, sink sensitivity, taint
      propagation, and the confirmation UX. `crates/executor` is blocked until
      this is reviewed.

### Security Demo (M1 = v0 DONE)

- [x] **REQ-quarantined-reader**: Quarantined reader worker producing
      genuinely-tainted typed extracts. *Done when:* the worker reads hostile
      input → emits a ValueNode; taint originates from the read Event (never
      hand-set).

- [x] **REQ-executor-stub**: `executor` stub — deterministic I2 interpreter.
      *Done when:* it walks the PlanNode DAG with I2 hardcoded and monotonic
      taint propagation through edges.

- [x] **REQ-mediated-sink-stub**: Mediated sink stub (e.g. `email.send`) with a
      sensitive `to` argument. *Done when:* the sink sensitivity map is
      hardcoded in v0 — no policy/schema system yet.

- [x] **REQ-approval-hook**: Approval hook for human confirmation of sensitive
      sink args. *Done when:* FAMP delivery; the prompt shows the literal value
      to confirm.

- [x] **REQ-s9-acceptance-test**: Automated integration test running the §9
      scenario end-to-end with a genuine taint chain — the single gate for v0
      DONE. *Done when:* (1) the reader emits a schema-valid typed extract as a
      ValueNode and the planner never sees the raw sentence; (2) taint is genuine
      — originates from the read Event, never hand-set at the sink; (3) a
      scripted plan (no LLM) flows that ValueNode into the sink's sensitive `to`
      argument; (4) the executor sees the recipient tainted
      (`external.untrusted`) in a sensitive sink arg → blocks; (5) it surfaces a
      literal-value confirmation prompt for the exact address; (6) the audit DAG
      shows the reader had no send cap, the sender never saw raw text, and an
      unbroken taint edge from the raw-read Event to the blocked sink argument.
      **Non-negotiable:** if taint is stapled on at the sink instead of
      propagated through the DAG, the demo fails — it proves nothing.

## Out of Scope

Explicitly excluded from v0 to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Cedar policy engine | Simple TOML/rules for sink access is fine; I2 stays in deterministic Rust |
| gVisor / Firecracker | bubblewrap + seccomp + Landlock is the v0 confinement boundary |
| LLM planner | A hard-coded / stub planner is sufficient to prove §9; LLM adds no security value |
| Cross-host delegation / Biscuit crypto | v3 concern; broker DB is the single-host authority for v0 |
| Web UI, marketplace, long-term memory, browser control | Out of the Intent-Runtime product boundary for v0 |
| Natural-language policy authoring | Not needed to prove I2; sink sensitivity is hardcoded in v0 |
| Mac / WSL2 support | All v0 security claims are Linux-only; deferred post-v0 best-effort |
| Broad effect taxonomy | Grow the ontology from audit-DAG observations, not upfront speculation |
