<objective>
Add the broker-side `process.exec` sink module: spawn the `caprun-exec-launcher`
(32-03) as a confined child, capture its stdout/stderr, bound it with a wall-clock
timeout and an output byte cap, and record the two-phase durable audit
(`process_exited` / `process_spawn_failed`). The mint of the captured output and
its return to the worker are 32-05.

kernel-confined subprocess spawn with captured output. It has no complete existing
template (file.create is a single syscall; the planner spawn does not capture).

`process_spawn_failed` Events; the tokio `process` feature on brokerd.

<locked_decisions>
Carried from Phase-32 research (grounded in live code) — capture verbatim so the
executor and a later reviewer see them:
1. **args encoding:** `process.exec`'s `args` is ONE `PlanArg` whose resolved
   literal is a JSON-serialized `Vec<String>` (`serde_json::to_string`). This
   module deserializes it broker-side (AFTER I2 cleared the value) and passes each
   element as a distinct `Command::args()` slot — NEVER a shell join, NEVER `sh -c`.
   The whole JSON blob is one taint-tracked unit. Stays inside the locked
   `PlanNode{sink,args}` shape (no new PlanNode shape).
3. **Async spawn/capture/timeout:** use `tokio::process::Command` (the tokio
   `process` feature) for a cancellable child; do NOT use `wait_with_output()` +
   `tokio::time::timeout` (it consumes `self` and leaks the child on timeout). Kill
   via the cancellable `Child` on wall-clock timeout (RESEARCH Pattern 1 / Pitfall 5).

**process_exited ownership (reconciles DESIGN §2.1 vs §2.4, resolved by the
RESEARCH system diagram, lines 191-199):** THIS sink module appends BOTH audit
events — `process_exited` on success (the mint-root + exit record) and
`process_spawn_failed` on failure — mirroring invoke_file_create's
`sink_executed`/`sink_execution_failed` two-phase shape exactly. `mint_from_exec`
(32-05) does NOT append its own event; it roots the minted value's
`provenance_chain[0]` on THIS module's `process_exited` event id. One event, both
roles — the strongest non-stapling guarantee (mirrors mint_from_read's file_read).
</locked_decisions>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| broker → confined child | The broker spawns an arbitrary target as a child and reads its piped output. |
| child output → broker memory | Captured stdout/stderr bytes cross into broker-owned memory (to be minted untrusted in 32-05). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-32-12 | Denial of Service | idle/sleeping child never exits | high | mitigate | wall-clock tokio::time::timeout + child.kill (RLIMIT_CPU alone does not bound wall time) |
| T-32-13 | Denial of Service | unbounded captured output exhausts broker memory | high | mitigate | combined-output byte cap, fail-closed (never fail-open unbounded) |
| T-32-14 | Denial of Service (resource leak) | leaked child/thread on timeout | medium | mitigate | cancellable tokio::process Child, not consuming wait_with_output (Pitfall 5) |
| T-32-15 | Repudiation | a spawn failure leaves no durable trace | medium | mitigate | two-phase `process_spawn_failed` appended before propagating; no silent retry |
| T-32-16 | Elevation of Privilege | broker spawns the worker/target unconfined | critical | mitigate | spawns only `caprun-exec-launcher`, which self-confines before its execve (32-03) |
</threat_model>
