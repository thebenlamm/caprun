<objective>
Create the `caprun-exec-launcher` helper binary (DESIGN §1.3 Option B): a sibling
crate, spawned unconfined by the broker (32-04), that self-confines post-fork
using the 32-02 primitives and then `execve`s the target command in its own
process image.

Purpose: The confined worker cannot `execve` (seccomp deny) and an arbitrary
child cannot self-confine after a handshake it doesn't have — so confinement must
be applied BEFORE the target's execve. Option B does this in the launcher's own
address space (the proven `apply_confinement()` ordering), avoiding any
`pre_exec`-closure async-signal-safety hazard (DESIGN §2.5, retired on this path).
Output: `cli/caprun-exec-launcher/{Cargo.toml,src/main.rs}`; workspace membership.
</objective>
