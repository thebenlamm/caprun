<objective>
Implement POLICY-03: the broker binds the session policy at session creation from a trusted source provably outside the confined worker's reach, makes it immutable for the session, and records its identity/hash as a genuine audit-DAG event. `bind_policy` (in brokerd) runs the SAME shared `refuse_if_beneath_workspace` containment check as MAC-key custody (Plan 02) against the policy path — refusing to run if it resolves at-or-beneath the workspace root (the F1-precedent attack) — then loads + parses the policy and returns it with its SHA-256 hash. The `caprun run` entrypoint binds it, appends a `policy_bound` event chained after `session_created`, and passes the immutable bound value into `run_broker_server` (replacing Plan 03's internal placeholder). A negative test proves a mid-session policy-file rewrite does NOT change the enforced allowlist.

Purpose: POLICY-03 is the converged BLOCKER — a policy loaded from a worker-writable location lets the worker widen its own allowlist. Binding from a trusted, contained source + immutability + a hash-chained audit event is what makes the trust surface sound. Reusing the shared helper (not a second copy) is the gate-record MAJOR-2 mandate, now exercised at BOTH call sites so the Plan-02 anti-drift gate covers the binder.

Output: `crates/brokerd/src/policy.rs` (the binder + tests); `run_broker_server` taking the policy as a parameter; `main.rs` binding + auditing + threading the policy; and the worker-rewrite immutability negative test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| trusted policy source → broker | The policy is bound from a trusted on-disk path (or a broker-constructed in-memory default) at session creation, refused if it resolves at-or-beneath the workspace root. |
| confined worker → policy file | The worker — the PRIMARY adversary — must not be able to change the enforced allowlist by rewriting a file; the bound policy is immutable and never re-read. |
| policy identity → audit DAG | The bound policy's hash is a genuine hash-chained event, provable after the fact. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-42-11 | Elevation of Privilege | policy from a worker-writable location (F1-precedent attack) | critical | mitigate | bind_policy runs the shared refuse_if_beneath_workspace against the policy path — a path at-or-beneath the workspace root refuses to run (no session). Reuses the Plan-02 helper, so the anti-drift gate covers BOTH call sites (DESIGN §5.3, §6 row 10). |
| T-42-12 | Tampering | mid-session policy rewrite | critical | mitigate | The policy is captured by value at bind time and threaded immutably; nothing re-reads the file. The immutability negative test proves a worker rewrite does not change enforcement (DESIGN §5.3, §6 row 10). |
| T-42-13 | Repudiation | unprovable enforced policy | high | mitigate | The policy identity/hash is a genuine append_event chained into the SHA-256 chain; verify_chain proves it after the fact (DESIGN §5.3). |
| T-42-14 | Denial of Service | unresolvable/unparseable policy path | medium | mitigate | Fail-closed: a specified-but-unresolvable/at-or-beneath/unparseable policy path is a hard Err — no session runs (DESIGN §4 policy-source row). |
