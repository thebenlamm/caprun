<objective>
Execute the DESIGN §5.3 / gate-record MAJOR-2 mandate: EXTRACT the inline F1 containment predicate (currently in `cli/caprun/src/key.rs::load_or_create_key`, `pub(crate)` and unreachable from the broker-side binder) into ONE shared, unit-tested helper `refuse_if_beneath_workspace(path, workspace_root) -> anyhow::Result<()>` that BOTH the MAC-key custody path AND (in Plan 04) the broker policy binder call. Rewire `load_or_create_key` to delegate to it, and add an anti-drift invariant so a divergent second copy cannot silently reappear.

Purpose: POLICY-03 requires the broker to bind the session policy from a trusted source refused if it resolves at-or-beneath the workspace root (the F1-precedent attack: the confined worker — caprun's PRIMARY adversary — could RequestFd any file beneath the workspace root and rewrite its own policy). The gate-record MAJOR-2 finding proved "reuse F1 from key.rs verbatim" is NOT factorable — the check is inline and `pub(crate)` in the CLI crate, so the brokerd binder cannot reach it. This plan makes it reachable and shared without changing the (already component-wise-correct) predicate logic.

Output: `crates/adapter-fs/src/containment.rs` (the shared helper + canonicalize-existing-or-parent + tests); `load_or_create_key` rewired to call it; an anti-drift gate in `check-invariants.sh`. The crate choice is adapter-fs because runtime-core is I/O-forbidden (Gate 2 bans `std::fs`), adapter-fs already owns `WorkspaceRoot` containment, and BOTH cli/caprun and brokerd already depend on it.
</objective>

| cli crate ↔ broker crate | The F1 predicate must be callable from BOTH; it is lifted to adapter-fs (a shared dependency) because runtime-core is I/O-forbidden. |
