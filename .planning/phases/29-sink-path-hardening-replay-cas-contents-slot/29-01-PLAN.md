    - Sink-scoping: same args, different `sink.0` => different key.

    Add `pub(crate) fn plan_node_idempotency_key(sink: &runtime_core::SinkId, args: &[runtime_core::PlanArg]) -> String`. Compute the DESIGN §c pinned key: `SHA256( sink.0 || sorted( (arg_name, value_id) pairs ) )`. Sort a `Vec<&PlanArg>` by `arg.name` before hashing (this is the load-bearing order-invariance step — mirror `combined_digest`'s `sort_by(|a,b| a.0.cmp(b.0))`). Feed `sink.0.as_bytes()`, then for each sorted arg feed `arg.name.as_bytes()` and `arg.value_id.0.to_string().as_bytes()` into one `Sha256` hasher; return `hex::encode(hasher.finalize())`.
