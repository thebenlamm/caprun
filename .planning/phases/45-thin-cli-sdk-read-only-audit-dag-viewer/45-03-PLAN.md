<objective>
Add a read-only `caprun audit <session_id> <audit-db-path>` viewer verb that opens the audit DB by path, renders the session's events/decisions + a `verify_chain` verdict to the terminal with EVERY literal control-char-neutralized (WG-2 / U1 M3), and loads the MAC key via a load-ONLY `load_existing_key` that fails closed on an absent key and refuses `:memory:` (WG-4 / U1 M2).

Purpose: This is the "INSPECT the proof" half of the trust surface — the design partner's read-only window into the audit DAG. It reuses `verify_chain` (audit.rs:1240), `query_events_by_session` (audit.rs:1056), the shared neutralizer (45-02), and the F1 containment refusal — adding no new TCB. The security-load-bearing part is fail-closed key custody: `load_or_create_key` CREATES a key when absent and returns a fresh key for `:memory:`, both of which would make a `verify_chain` verdict meaningless, so the viewer needs a load-only sibling.

Output: A read-only `caprun audit` verb + `load_existing_key` + a viewer test suite.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| audit DB on disk → viewer | the viewer reads a possibly-attacker-influenced audit DB (tainted literals in event payloads) and must never execute or be spoofed by its bytes. |
| MAC key custody → viewer | the viewer must verify the chain against the REAL signing key or refuse — never a fresh/`:memory:` key (fail-closed). |
| confined worker → audit DB / key | the audit DB + `.key` must be out of the confined worker's reach (F1), else a forged chain could verify. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-45-07 | Spoofing | a fresh/`:memory:` key yields a green-but-meaningless `verify_chain` verdict | critical | mitigate | `load_existing_key` load-only: refuse `:memory:`, hard-error on absent key, never create (WG-4 / U1 M2, Pitfall 1). |
| T-45-08 | Tampering | terminal ANSI/CR spoofing of audit lines via a tainted literal | high | mitigate | universal `neutralize_control_chars` on EVERY displayed field (all sinks, not git.push-only) via the 45-02 shared fn (WG-2 / U1 M3, Pitfall 2). |
| T-45-09 | Elevation | the viewer opens the workspace root / appends an event / invokes a sink → becomes an authority surface | high | mitigate | model on `review`: read-only conn (SQLITE_OPEN_READ_ONLY), no workspace root, no sink, no event, no state transition (Pitfall 4). |
| T-45-10 | Tampering | an audit DB beneath the workspace root lets the confined worker forge a chain the viewer then blesses | high | mitigate | the viewer runs the SAME F1 `refuse_if_beneath_workspace` check (via `load_existing_key`) key custody uses — refused, out of worker reach. |
</threat_model>
