<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator CLI → broker | the `caprun run` operator types the intent + optional `--policy` path; the intent literal is trusted ONLY because it is operator-typed. |
| file/stream/env source → broker | any ingested content is untrusted and must be minted TAINTED, never through the trusted intent mint (M7). |
| policy source → broker | the `--policy`/`CAPRUN_POLICY` path is bound via `bind_policy` with the F1 containment refusal (POLICY-03). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-45-01 | Elevation | file/env content laundered through `mint_from_intent` as trusted | high | mitigate | M7 disjointness: file/stream/env ingestion routes only through the existing `mint_from_read` taint-mint site; operator-literal path stays disjoint; check-invariants Gate 3 forbids a second mint site (Task 3). |
| T-45-02 | Tampering | a `--policy` path beneath the workspace root lets a confined worker widen its own allowlist | high | mitigate | the flag resolves to the SAME `bind_policy` call whose `refuse_if_beneath_workspace` F1 refusal already rejects at-or-beneath-workspace paths (POLICY-03, reused verbatim). |
| T-45-03 | Repudiation | a blocked effect_id is buried in worker Debug output; the operator cannot reach confirm/deny | medium | mitigate | the parent queries `pending_confirmations` read-only post-run and surfaces every Pending effect_id + review/confirm/deny pointer (WG-5). |
| T-45-04 | Tampering | supply-chain / TCB drift via a new mint or effect path | high | mitigate | no new crate, no new mint site, no raw `EffectRequest`; `scripts/check-invariants.sh` (Gate 1/3/5) run in verification. |
</threat_model>
