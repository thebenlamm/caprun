<objective>
Build the genuine prompt-injection channel that lets a hostile document's
embedded instruction reach the LLM planner's context (GATE-01), and the
deterministic construction-site sentinel assertion that proves sink-arg
literals never leak into that context (GATE-04).

Today (Phase 21) the LLM sees ONLY opaque handle-ID labels + slot hints —
there is no channel by which an attacker's injected instruction text reaches
the model, so "the planner complies with an injection" cannot yet be
demonstrated. This plan opens that channel the way real prompt injection
works: through content the system legitimately must show the model to do its
job (`task_instruction`), while the sensitive VALUES the instruction talks
about stay opaque handle IDs the LLM can only reference, never read.

Purpose: make the injection load-bearing (the LLM is offered both a trusted
and the tainted recipient handle; the injection is what routes the tainted
one to `to`) so Plan 22-02's live proof is genuine compliance, not a
mechanical slot mapping. The two-handle offering is keyed on the tainted
derived-recipient existing, DECOUPLED from the injection marker — this is what
lets 22-02 run a three-leg proof where a control leg (both handles offered, NO
injection) shows the model picks the TRUSTED handle, isolating the injection as
the causal factor rather than a positional bias. This plan also replaces the
retired context-dump grep with a deterministic sink-arg-literal-leak test.

Output: the `task_instruction` wire channel across llm-planner + the worker's
planner seam, and the GATE-04 sentinel unit test.
</objective>
