<objective>
Create the `llm-planner` pure library crate: the typed wire contract between the
worker-side `LlmPlanner` proxy and the out-of-process LLM sidecar, plus the
pure prompt/tool-schema construction and the fail-closed response validator.

This crate holds ZERO network code and ZERO reqwest dependency — it is the
handle-only, literal-free layer both the sidecar (Plan 02) and the worker-side
proxy (Plan 03) build on. Because it lives under `crates/*`, the existing
workspace-members glob picks it up with no root Cargo.toml change.

Purpose: Establish the structural guarantee (PLANNER-03 / PLANNER-04) that the
LLM can only ever reference opaque handle IDs, never a literal — enforced by
the wire types and the tool-schema `enum`, before any HTTP or process wiring
exists.

Output: `crates/llm-planner` with wire types, `build_planner_prompt`,
`build_tool_schema`, and `parse_planner_response`, all unit-tested on macOS.

| Boundary | Description |
|----------|-------------|
| worker-proxy → LLM sidecar (wire types) | The PlannerRequest crossing this boundary must never carry a literal; only handle IDs + slot hints. |
| LLM tool-call → PlanNode | Untrusted model output crosses here; parse_planner_response is the validation gate. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-21-01 | Information Disclosure | PlannerRequest / build_planner_prompt | high | mitigate | Wire types carry no literal field; build_planner_prompt reads only handle IDs + slot hints. Key-set serde test proves no literal-carrying field exists. |
| T-21-02 | Tampering | LLM tool-call output | high | mitigate | parse_planner_response fails closed on unknown sink / unknown handle / malformed JSON; never fabricates a handle. Tool schema `enum` constrains value_id to offered handles. |
| T-21-03 | Spoofing | fabricated handle in response | medium | mitigate | value_id must be a member of the caller-supplied `offered` set; not-a-member → Err. |
</threat_model>
