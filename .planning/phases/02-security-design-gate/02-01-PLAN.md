  <action>
    Write `planning-docs/DESIGN-taint-model.md` as a formal security spec (not project notes). Use MUST / MUST NOT / is REQUIRED for every invariant — never "should" / "typically" / "generally" for hard requirements. Include these nine sections with these exact headings and content, sourced verbatim where the research marks [VERIFIED]:

    1. "## Invariant Statements" — state I1 (no LLM context may simultaneously hold untrusted/attacker-controlled content AND authority to cause an irreversible or external side effect) and I0 (a Session whose intent text or seed derives from external/untrusted content starts draft-only and cannot auto-authorize Tier 3+ effects; human gate required on context creation from tainted data). Cite source as planning-docs/PLAN.md §Security Model.
    2. "## Default Taint Model — Dynamic Taint" — state the default is dynamic taint: any context MAY view raw untrusted bytes, and the instant it does it becomes tainted and drops to draft-only for the remainder of that context. State the rationale: typed+lossy extraction kills legitimate work needing raw content; dynamic taint allows raw viewing but restricts the context to draft-only. State the hard split is reserved for Tier 3+ tasks only.
    3. "## High-Risk Mode — Hard Planner/Worker Split (Tier 3+)" — define the privileged planner (sees user request, schemas, summaries, metadata, typed extracts; never raw hostile content; holds planning authority not execution authority) and the quarantined worker (reads hostile docs/email/web; holds NO send/delete/spend/deploy/secret/network caps; emits only schema-validated typed lossy non-instructional extracts). State the trigger condition: task touches Tier 3+ sinks.
    4. "## Trust Tier Definitions" — enumerate Tier 0 (Observe), Tier 1 (Draft/dry-run/read), Tier 2 (Reversible local mutation), Tier 3 (External side effect, bounded blast radius), Tier 4 (Money / deletion / production / identity / irreversible-send). State: human stays out for Tier 0-2; enters at Tier 3 when novel/high-risk; always at Tier 4 unless an explicit standing policy with hard ceilings pre-permits. Use the locked tiers from PLAN.md — do not invent a new tier system.
    5. "## Worker Output Contract" — describe the typed, lossy extract the quarantined worker emits: fields source, taint (label list), claims (typed value + evidence_span_hash + confidence), stripped (instructions_to_agent, tool_requests). State that schema validation checks shape, not truth: a well-shaped value whose content is attacker-chosen is still attacker-controlled, and this is the gap the plan executor (I2) closes. Present the contract as a short illustrative field list, NOT a full fenced file.
    6. "## Taint Label Vocabulary (v0)" — define the minimal label set needed for §9 (RESOLVES open research question 1: minimal set, not a full enum). Define external.untrusted (value originated from content outside the trust boundary) and email.raw (raw email body read by quarantined worker). State that v0's sink sensitivity map is hardcoded — no Cedar/schema taint policy system — and that label extensibility is a post-v0 concern.
    7. "## Threat Model — I1 Attack Surface" — include the two-attack-mode distinction explicitly. Instruction injection (hostile content tells the LLM to perform a dangerous effect): DEFEATED by the planner/worker split because the reading LLM holds no dangerous cap. Value injection (hostile content supplies a value the planner legitimately uses, riding a valid schema straight to the sink): NOT DEFEATED by the split alone; defeated by I2 (the plan executor, specified in DESIGN-plan-executor.md). State explicitly: "The planner/worker split is necessary but not sufficient." Also include I0 injection (the session/intent itself seeded from hostile content): defeated by the I0 draft-only rule.
    8. "## Accepted Residual Risks" — document, as accepted for v0: steganographic encoding in extract values (typed+lossy mitigates but does not eliminate); planner/intent-creation injection (mitigated by I0 draft-only + human gate on Tier 3+ from tainted session seeds); fd revocation after SCM_RIGHTS handoff (mitigated by disposable workers + mediated path).
    9. "## Genuine-Taint Requirement & I0 Acceptance Predicate" — state the genuine-taint requirement: taint MUST originate from a read Event recorded in the audit DAG and MUST NOT be hand-set at the sink; the §9 acceptance test fails if taint is stapled on at the sink instead of propagated. State the I0 Done-When predicate: a Session whose intent text or seed derives from external/untrusted content (a) starts draft-only AND (b) cannot auto-authorize Tier 3+ effects, with a human gate required.

    Cite prior art briefly: note that this design converges on CaMeL (arXiv 2503.18813) P-LLM/Q-LLM split and FIDES (arXiv 2505.23643) information-flow control, with AgentOS adding the audit-DAG genuine-taint requirement. Reference REQ-design-taint-model in the doc header. Do NOT create or reference any code under crates/executor.
