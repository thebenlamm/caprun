<objective>
Promote the three Phase-15 genuine-taint proof predicates —
`assert_unbroken_edge`, `genuine_derivation_binds`, and their helper
`union_provenance_chains` — out of the private brokerd test binary
`crates/brokerd/tests/extract_provenance_threading.rs` and into a public
`brokerd::provenance_proof` module, so Phase 17's live composed test (in a
DIFFERENT crate, `cli/caprun/tests/`) can RE-RUN the exact same HARD-GATE
check against the live run's anchors instead of reimplementing it.

Purpose: The coordinator's assertion tooth #2 (COORD-T2) requires re-proving
genuine taint descent "IN THIS composed run" using
`assert_unbroken_edge`/`genuine_derivation_binds` "from
crates/brokerd/tests/extract_provenance_threading.rs". Cross-crate test
modules are NOT importable, so the live test in `cli/caprun` cannot reach these
today. Reimplementing them is explicitly forbidden (17-RESEARCH.md "Don't
Hand-Roll": reimplementing risks silently weakening the exact check the
milestone depends on). Promotion to one public source of truth is the only
correct resolution.

Output: A `brokerd::provenance_proof` public module; the Phase-15 test
refactored to consume it (so there is one implementation, exercised by both
the existing DB-alone proof and the new live proof).
</objective>
