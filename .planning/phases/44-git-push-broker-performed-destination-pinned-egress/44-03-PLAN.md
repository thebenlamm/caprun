<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| broker → external host | the two outbound git-smart-HTTP requests (info/refs GET + receive-pack POST) cross the trust boundary through one frozen-IP redirect-none client. |
| broker env → transfer | the push credential is read from broker-local process env only and set on the single POST. |
| workspace `.git` → confined pack-gen child | an untrusted-worker-writable `.git` is read by the net-denied, Landlocked, config-neutralized pack-objects child. |
| transfer → audit chain / value store / logs | only OPAQUE scrubbed events cross into the audit DAG; nothing is minted; no credential/URL survives into logs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-44-09 | Elevation | a net-allowed pack-gen child = arbitrary credential-bearing egress | critical | mitigate | pack-gen runs under exec_child_filter UNCHANGED (AF_INET/AF_INET6 EPERM); the pin is application-layer (broker reqwest), NEVER seccomp; the child never opens a network socket (DESIGN §1.2, §1.8 backstop). |
| T-44-10 | Information disclosure | credential / remote-URL leaked to value store, audit, or broker LOG | critical | mitigate | credential broker-env-only, never a plan arg/ValueNode/audit literal; a scrub helper strips token/userinfo-URL/proxy-auth from every logged string; opaque `git_push_*` events; captured output never minted (DESIGN §1.4 MINOR-4). |
| T-44-11 | Tampering | git-config RCE via the untrusted workspace `.git/config` during pack-gen | high | mitigate | pack-gen child clones git.commit's neutralization (`-c core.hooksPath=/dev/null`, GIT_CONFIG_NOSYSTEM=1, GIT_CONFIG_GLOBAL=/dev/null) + Landlock-to-workspace + env_clear (RESEARCH A14). |
