<objective>
Collapse the broker's dual dispatch into one stateful `brokerd::server` path and make it the live, session-scoped, fail-closed reference monitor. This plan: (1) changes `run_broker_server` to a session-bound, per-connection-stateful form and drops the shared `ValueStore`; (2) wires `RequestFd` (fd-pass), `ReportClaims` (genuine-taint mint), and a durable fail-closed `sink_blocked` `SubmitPlanNode` handler into one dispatch; (3) removes the request-supplied `session_id` from `SubmitPlanNode`; and (4) updates the `uds_ipc.rs` call sites to the new signature. This is the security-critical core of Phase 5.

Purpose: ASM-01 (single dispatch), ASM-02 (executor runs live; placeholder gone broker-side), ASM-04 (mint_from_read anchored), HARD-03 (session-scoped handles + untrusted session_id removed), ACC-02 (durable causal sink_blocked, fail-closed).
Output: A rewritten `server.rs`, the breaking proto change, updated legacy test call sites, and a new cross-platform dispatch test file.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| confined worker → broker (IPC) | Worker supplies claims, plan nodes, and (formerly) a session_id; the broker is the reference monitor that must not trust message-supplied identity |
| broker → audit DAG (SQLite) | Security decisions must be durably recorded before they are reported as having occurred |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-03 | Spoofing | SubmitPlanNode session identity | high | mitigate | Remove session_id from the IPC message; evaluate against the connection-established session_id only (Task 1) |
| T-05-04 | Elevation of Privilege | cross-session handle resolution | high | mitigate | Per-connection ValueStore::default(); a handle from another connection resolves None → Denied (Task 1 + Task 3 test) |
| T-05-05 | Tampering | taint stapling at the sink | critical | mitigate | mint_from_read is the only mint call on the live path; anti-stapling grep gate in Task 2; provenance-anchor test in Task 3 |
| T-05-06 | Repudiation | audit evasion via best-effort append | high | mitigate | Fail-closed `?` propagation on sink_blocked append; decision sent only after durable write (Task 2 + Task 3 test) |
| T-05-07 | Information Disclosure | raw bytes via claims | medium | mitigate | ReportClaims carries only typed extracts; mint uses the claim value, never a raw sentence |

No package-manager installs in this plan (zero new dependencies) — supply-chain checkpoint N/A.
</threat_model>
