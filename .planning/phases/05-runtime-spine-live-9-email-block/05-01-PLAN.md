<objective>
Add the typed IPC vocabulary for the worker→broker claims protocol: a bounded `WorkerClaim` enum (Phase 5 ships the `EmailAddress` variant only), a `ReportClaims` request variant, and a `ClaimsReceived` response variant. All changes are ADDITIVE so the workspace stays build-green after this wave — the breaking proto changes (removing `session_id` from `SubmitPlanNode`, retiring `ReportRead` handling) are made in Plan 02 alongside the server rewrite that depends on them.

Purpose: ASM-03 requires the confined worker to emit typed claims (not raw bytes) and the broker to fail closed on unknown claim kinds. This plan establishes the wire types and proves the fail-closed serde contract before any handler consumes them.
Output: New protocol types in `proto.rs` plus a cross-platform serde round-trip + fail-closed test file.
</objective>
