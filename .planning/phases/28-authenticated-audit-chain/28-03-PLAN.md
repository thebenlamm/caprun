<objective>
Convert the audit hash chain from an unkeyed SHA-256 self-consistency check into a keyed HMAC-SHA256 authenticated chain, threading the broker key to every production `append_event` call site and both `verify_chain` callers, and wiring the run path to load the key (F1-checked) at startup. This is the single atomic signature-change pass (RESEARCH step 4, Phase-27 "one signature-change pass" discipline).

Purpose: This delivers HARDEN-02 Success Criteria 1 and 2 directly — a bare `events`-table writer cannot forge a self-consistent chain because they lack the key. (Tail-truncation detection via the anchor is Plan 04; `pending_confirmations` fold + confirm/deny gate is Plan 05.)
Output: keyed `compute_event_hash`, threaded key across 19+2 sites, runtime key wiring, forgery-rejection + key-dependence tests.
</objective>
