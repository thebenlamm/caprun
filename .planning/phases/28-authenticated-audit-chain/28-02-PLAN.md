<objective>
Create the shared cross-process MAC-key custody helper and the F1 fail-closed startup refusal, unit-tested in isolation BEFORE any call site consumes the key.

Purpose: HARDEN-02's entire authenticity guarantee rests on (a) a stable secret key shared across the single-shot `caprun run` process and the later, separate `caprun confirm`/`deny` process — a per-process fresh key would make every legitimate confirm fail `DigestMismatch` (Pitfall 2) — and (b) the DESIGN §b F1 BLOCKER: the key file must be provably outside the confined worker's `read_within`-reachable set, broker-enforced, never operator convention. Isolating the helper + its F1 negative test here (RESEARCH recommended sequencing step 3) means a later red test unambiguously indicts the wiring, not the custody logic.
Output: `load_or_create_key` (with F1 containment) + its unit tests. NOT yet wired into the runtime `open_audit_db` flow — Plan 03 wires the run path, Plan 05 the confirm/deny path.
