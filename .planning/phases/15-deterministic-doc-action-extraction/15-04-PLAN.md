## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| hostile doc bytes -> confined worker | Multi-fragment extraction + concat run over hostile bytes inside the Landlock/seccomp sandbox; the transform must not leak into the broker control plane. |
| worker transform -> mint | The worker transforms its OWN extracted fragment values (never a resolved broker literal reused as the same handle); a fresh derived handle is obtained before use as a plan-node arg. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-15-13 | Elevation of Privilege | worker extraction locus | high | mitigate | Extraction + concat run worker-side (confined), emitting only typed claims + plan nodes — EXTRACT-01. |
| T-15-14 | Tampering (drift) | mint-after-transform | high | mitigate | The worker mints (via ReportDerivedClaim) AFTER transforming, and never transforms after minting or reuses a pre-transform handle — DESIGN-confirm-binding.md D-08. |
| T-15-15 | Information Disclosure | lossy guarantee | medium | mitigate | Only typed fragment tokens + the transformed literal cross IPC; the raw hostile sentence is discarded worker-side. |
| T-15-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package installs in this plan (concat transform is String-only; base64 deferred) — no legitimacy checkpoint required. |
