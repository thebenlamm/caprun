<objective>
Build the phase's HARD-GATE proof: a programmatic, DB-alone audit-DAG query that verifies genuine taint propagation — including through a concatenation transform — for EVERY blocked arg in a multi-anchor Block, paired with an anti-staple control that rejects a fabricated/re-anchored chain. Plus the realistic hostile-doc fixture (CONFIRM-02) the proof consumes.

Purpose: This is the single non-negotiable invariant of the whole milestone (CLAUDE.md #1, ROADMAP Phase 15 success criterion 3: "phase FAILS if not met"). "Taint stapled at the sink proves nothing." A passing proof on genuine data is worthless without a paired FAILING proof on fabricated data — so this plan builds both. Because Phase 12 mandates collect-then-Block, the proof must hold for BOTH the tainted derived recipient AND the tainted body in one Block, not just one (D-16 per-arg).

Output: `find_event_by_id`, the new EXTRACT-02/03 test file, and the CONFIRM-02 fixture. Runs cross-platform (SQLite alone, no Linux sandbox) modeled on durable_anchor.rs's after-exit DB-alone pattern.
</objective>

| Boundary | Description |
|----------|-------------|
| persisted audit DAG -> verification query | The EXTRACT-02 proof reads the reopened SQLite DAG as ground truth; it must not trust in-memory ValueStore state (which does not survive process exit) — only durable events + persisted anchors. |
| genuine vs. fabricated provenance | The proof's entire value is its ability to tell a threaded chain from a stapled one; a proof that passes on both is worse than none. |
