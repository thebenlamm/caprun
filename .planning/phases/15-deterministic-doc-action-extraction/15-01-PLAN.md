<objective>
Build the provenance-threading mint primitive that closes the milestone's #1 laundering BLOCKER, plus the confined-worker-callable multi-fragment extraction and concatenation helpers that feed it.

Purpose: A transform (concatenating two doc fields into a recipient) must NOT re-anchor the derived value's lineage at a fresh event. Copying the taint LABEL forward while rooting the derived value at a brand-new event is "taint stapled on at the sink" — it satisfies the letter of "an unbroken edge exists" while proving nothing (CLAUDE.md hard constraint #1; DESIGN-confirm-binding.md "Provenance-Threading for Transform-Derived Mints", D-16). This plan makes a genuinely-threaded mint the only mint a derived value can pass through.

Output: `mint_from_derivation` (thread inputs' chains, fail closed on a re-anchored chain), a `doc_fragment` raw claim type, and hand-rolled fragment-scan + concat helpers. Consumed by Plan 02 (the DB-alone EXTRACT-02 proof) and Plan 03/04 (the live confined-worker path).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| confined worker -> broker mint | The worker supplies an already-transformed literal + input handles; the broker mints. The transformed bytes are untrusted (tainted) and will be blocked; the broker must not trust their provenance except as threaded from the inputs it holds. |
| raw doc bytes -> extraction helpers | Hostile bytes flow into extract_doc_fragments / concat; a parser bug here is attack surface, so it stays a hand-rolled, dependency-free scanner. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-15-01 | Spoofing (of provenance) | mint_from_derivation | critical | mitigate | provenance_chain is the concat of inputs' read-rooted chains; a fresh-rooted/re-anchored derived value is impossible by construction and zero-input mints fail closed (D-16). |
| T-15-02 | Tampering | mint_from_read | high | mitigate | mint_from_read is unchanged except an additive claim_type arm; derived values never mint through it. |
| T-15-03 | Elevation of Privilege | extraction/transform helpers | high | mitigate | Extraction + concat are pure, hand-rolled, dependency-free (no regex/parser lib); the broker never re-applies the transform. |
| T-15-04 | Tampering | derivation Event linkage | medium | mitigate | The derivation event's multi-input lineage rides only in provenance_chain (value-lineage graph), never conflated into single-parent Event.parent_id (two-graphs-never-share-edges). |
