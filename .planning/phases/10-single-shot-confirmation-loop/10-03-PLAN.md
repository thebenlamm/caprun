enforced by re-reading the persisted DB row (never in-memory) on every invocation.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| human operator CLI (`caprun confirm`/`deny`) → broker TCB | The only surface that releases a blocked effect; untrusted CLI `effect_id` input must fail closed. |
| confined worker/planner → confirm | A worker/planner must not be able to forge a pending confirmation; effect_id is broker-minted at Block time. |
| persisted PendingConfirmation.state → decision | Terminal state is re-read from the DB on every invocation, never from memory. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-01 | Tampering/EoP | replay: re-confirm an already-terminal effect_id | high | mitigate | `confirm`/`deny` re-read `PendingConfirmation.state` from the DB each call and refuse any non-Pending state (`AlreadyTerminal`, exit 5); `transition_state`'s SQL `AND state='pending'` guard closes the race — proven by the cross-process spawn-twice test (CONFIRM-03). |
| T-10-03 | Spoofing | forged effect_id supplied to `caprun confirm` | high | mitigate | effect_id is broker-minted at Block time (server.rs `Uuid::new_v4()`), never client/worker-supplied; `find_pending_confirmation` returns `None` for any id never persisted → `UnknownEffect`, exit 4 (CONFIRM-02/04). |
| T-10-04 | Repudiation | human confirms the wrong literal via a truncated display | medium | mitigate | `render_block_display` shows the byte-exact literal verbatim (no truncation/elision) plus taint + provenance_chain (CONFIRM-01). Path-specific confusable canonicalization is DESIGN Accepted Residual Risk 1. |
| T-10-05 | EoP | confirm bypasses/re-runs I2 | high | mitigate | confirm calls `invoke_file_create_from_resolved` on the frozen snapshot and NEVER `executor::submit_plan_node`; no allowlist/standing policy is created or consulted (CON-i2-non-bypassable, Single-Shot Release Semantics). Verified by grep gate + a "second block in a new session still blocks" argument. |
| T-10-09 | Information Disclosure | releasing a literal whose blocked_literals row was redacted | medium | mitigate | confirm's redaction gate: `get_blocked_literal` `None` → `BlockedLiteralRedacted`, refuse to release (exit 6) — Pitfall 4 fail-closed. |
| T-10-06 | Tampering | workspace changed between Block and Confirm (TOCTOU) | low | accept | `O_EXCL` + `RESOLVE_BENEATH`/`RESOLVE_NO_SYMLINKS` still fail closed at confirm-invocation time (DESIGN Accepted Residual Risk 2); surfaces as `ConfirmedButSinkFailed` (exit 3). |
| T-10-SC | Tampering | package installs | n/a | accept | No new dependencies this phase (RESEARCH Package Legitimacy Audit). |
</threat_model>
