<objective>
Wire the durable checkpoint into the block path and build the frozen-snapshot sink primitive:
(1) at the moment `executor::submit_plan_node` returns `BlockedPendingConfirmation`, resolve the FULL
arg set from the still-live per-connection `ValueStore` into a `Vec<ResolvedArg>` snapshot and persist
a `PendingConfirmation` row atomically alongside the `sink_blocked` event; and
(2) add `invoke_file_create_from_resolved`, a `ValueStore`-free sibling of `invoke_file_create` that
re-invokes the `file.create` sink from frozen `ResolvedArg` literals — the function `caprun confirm`
will call in Plan 03.

Purpose: the in-memory `ValueStore` that can resolve the block's `ValueId` handles is gone once the
blocking process exits (`DESIGN-confirmation-release.md` "The Problem Being Solved"). Block time is
the only moment the full literal set exists; this plan captures it durably and builds the invocation
path that consumes the frozen snapshot without ever re-resolving or re-submitting.

Output: `WorkspaceRoot::root_path()`, the block-arm persistence wiring in `dispatch_request`, and
`invoke_file_create_from_resolved`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| block-time ValueStore → durable snapshot | The only moment the block's full literal set is recoverable; captured here or lost at process exit. |
| confirm-time frozen literals → filesystem sink | Re-invocation uses adjudicated frozen literals, never a re-resolution or re-decision. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-02 | Tampering/EoP | orphaned block or orphaned checkpoint from a non-atomic write | high | mitigate | `insert_pending_confirmation` runs inside the same `conn.lock()` scope as `append_event(sink_blocked)` + `insert_blocked_literal`, all under fail-closed `?` — they commit together or the block aborts before responding (DESIGN Persistence contract). |
| T-10-05 | EoP | confirm-path re-invocation bypassing I2 by re-deciding | high | mitigate | `invoke_file_create_from_resolved` takes frozen `&[ResolvedArg]` and calls the sink adapter directly — it never constructs a ValueStore nor calls `executor::submit_plan_node`, so I2 is neither re-run nor bypassable (CON-i2-non-bypassable). |
| T-10-06 | Tampering | workspace changed between Block and Confirm (TOCTOU) | low | accept | `create_exclusive_within`'s `O_EXCL` + `openat2 RESOLVE_BENEATH/RESOLVE_NO_SYMLINKS` still fire at confirm-invocation time and fail closed on conflict (DESIGN Accepted Residual Risk 2). |
| T-10-SC | Tampering | package installs | n/a | accept | No new dependencies this phase (RESEARCH Package Legitimacy Audit). |
</threat_model>
