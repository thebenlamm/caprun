<objective>
Add the minimal, non-default cargo feature `mock-egress-ca` that lets Plan 40-03's mock GitHub endpoint be reached over REAL TLS by the broker egress while genuinely riding the shipped SSRF pin — without weakening the release build.

Purpose: LIVE-03 requires a live github.pr POST that "still rides validate_url/allowlist/resolve-and-pin" (40-CONTEXT decision 3) against a mock (decision 5). The shipped egress trusts ONLY compiled-in webpki-roots and allowlists ONLY api.github.com, so a mock cannot be reached without either a bypass (forbidden) or a test-only trust addition. This plan adds the latter: a feature-gated test CA + one feature-gated mock host, mirroring brokerd's existing `test-fixtures` feature pattern. The DESIGN doc anticipates exactly this — §11.3 notes "offline/mocked tests do not catch a cert-root regression," i.e. the mock uses a non-real cert and the REAL api.github.com GET (Plan 40-04) is what proves webpki-roots. This is TCB-adjacent egress code and MUST be scrutinized by the closing fresh adversarial code-trace.
Output: a feature that, when enabled, adds one trust anchor + one allowlist host; when disabled, changes nothing, asserted by a test.
</objective>
