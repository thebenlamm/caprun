Deliver the one genuinely-new mechanism of the milestone — `mint_from_http` — and wire the `http.request` GET end to end, per DESIGN §3.3 / §3.5 / §10 and 37-CONTEXT decisions 3, 5, 8. On an Allowed `Observe` GET, the broker fetches (Plan 02), mints the response body as untrusted-on-arrival rooted on a real `http_response_received` Event, and demotes the session to draft-only (I1). Getting the event-first→mint→demote ordering and the anti-staple provenance anchor right is the whole point — a taint stapled at the sink instead of propagated through the DAG proves nothing.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external network → broker | untrusted response bytes minted as a ValueNode |
| minted value → sink | a fetched value may be routed toward an irreversible sink arg |
| mint call site | only sanctioned loci may mint (Gate 3) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-37-02 | Elevation of privilege | fetched response routed into a sensitive sink (exfil/injection) | critical | mitigate | mint_from_http stamps [ExternalUntrusted, HttpRaw] with genuine, non-stapled provenance rooted on http_response_received; is_untrusted(HttpRaw) forces an I2 Block in any routing/content-sensitive slot (anti-staple test proves it) |
| T-37-05 | Spoofing | session not demoted on untrusted inbound (I1 hole) | high | mitigate | atomic in-conn Draft UPDATE + session_demoted Event in mint_from_http, AND the shared in-memory session_status cell written Draft in the server dispatch (RequestFd exemplar) — a later same-connection node observes Draft |
