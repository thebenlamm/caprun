<objective>
Add the shared workspace-root dirfd capability (read-side) that closes HARD-04: the broker's `RequestFd` handler currently calls `std::fs::File::open(&path)` on a path the confined worker fully controls (`crates/brokerd/src/server.rs:248-251`), so a compromised/injected worker can request any broker-openable absolute path. This plan introduces `adapter_fs::workspace::WorkspaceRoot(OwnedFd)`, opened ONCE in `main()`, and resolves all `RequestFd` reads beneath it via `openat2(RESOLVE_BENEATH | RESOLVE_NO_SYMLINKS)` — a single kernel syscall that is TOCTOU-safe and rejects absolute/traversal/symlink-escape paths.

This is the read-side prerequisite for SINK-04 (write-side `create_exclusive_within`, planned in 07-04). This plan builds ONLY the capability struct + `read_within` + the broker threading; the write method is out of scope here.

Purpose: eliminate the unrestricted broker-side open — the concrete HARD-04 vulnerability.
Output: `WorkspaceRoot` capability, `read_within`, and the `Arc<WorkspaceRoot>` threading through the broker server.
</objective>

  <action>
Create `crates/adapter-fs/src/workspace.rs` and register `pub mod workspace;` in `crates/adapter-fs/src/lib.rs`.

Define `pub struct WorkspaceRoot` holding the anchor dirfd (`OwnedFd`) plus the root `PathBuf` (the latter is used only by the non-Linux stub). Add a constructor `WorkspaceRoot::open(root: &Path) -> std::io::Result<Self>` that obtains the dirfd via `nix::fcntl::open(root, OFlag::O_DIRECTORY | OFlag::O_RDONLY, Mode::empty())` — plain `open`, NOT `openat2`; the broker legitimately has ambient access to the root, and this one call establishes the anchor. Map the `nix` error to `std::io::Error` (`std::io::Error::from_raw_os_error(e as i32)`); if `nix::fcntl::open` returns a `RawFd` on this nix version, wrap it via `OwnedFd::from_raw_fd` (unsafe, valid fd postcondition), otherwise use the returned `OwnedFd` directly.

Add `read_within` as a cfg-gated pair mirroring `sandbox::landlock::deny_all_filesystem`:
- `#[cfg(target_os = "linux")] pub fn read_within(&self, rel_path: &str) -> std::io::Result<std::fs::File>`: build `OpenHow::new().flags(OFlag::O_RDONLY).resolve(ResolveFlag::RESOLVE_BENEATH | ResolveFlag::RESOLVE_NO_SYMLINKS)`, call `nix::fcntl::openat2(&self.dirfd, rel_path, how)`, map `nix::Error` to `std::io::Error::from_raw_os_error(e as i32)` (a `RESOLVE_*` violation surfaces as `EXDEV`), and return `std::fs::File::from(owned_fd)`. Both `RESOLVE_BENEATH` and `RESOLVE_NO_SYMLINKS` are required — `RESOLVE_BENEATH` alone does NOT block in-tree symlink traversal (RESEARCH Q1 caveat, Pitfall 2).
- `#[cfg(not(target_os = "linux"))] pub fn read_within(&self, rel_path: &str) -> std::io::Result<std::fs::File>`: no security claim on non-Linux — `std::fs::File::open(self.root_path.join(rel_path))`. This exists solely so the workspace crate compiles on the Mac dev machine (CLAUDE.md scope).

Do NOT add `create_exclusive_within` / any write path — that is SINK-04, planned in 07-04. Import `std::os::fd::{AsFd, FromRawFd, OwnedFd}` and `nix::fcntl::{OFlag, OpenHow, ResolveFlag, openat2, open}` / `nix::sys::stat::Mode` as needed under the correct cfg gates.

Add an inline `#[cfg(test)] mod tests` block. Enforcement tests are `#[cfg(target_os = "linux")]`-gated at the fn level (they exercise real openat2 semantics; on macOS they report 0-passed, which is expected — CLAUDE.md). Each test creates a `tempfile::tempdir()` root, opens a `WorkspaceRoot` on it, and asserts:
  - a legit relative read (a real in-root file) returns `Ok` and yields the expected bytes;
  - an absolute path arg (e.g. `/etc/hostname`) returns `Err` (RESOLVE_BENEATH rejects; raw OS error `EXDEV`);
  - a `../` traversal path (a file placed in the tempdir's PARENT, requested as `../<name>`) returns `Err`;
  - a symlink inside the root pointing to a target OUTSIDE the root returns `Err` (RESOLVE_NO_SYMLINKS rejects at resolution — the read must fail, NOT follow-then-block).
Use `tempfile` if already a dev-dependency of adapter-fs; if not, use `std::env::temp_dir()` with a uniquely-named subdir created/cleaned by the test.
