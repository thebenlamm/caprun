## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| executor decision → durable audit DAG | The block decision must persist as tamper-evident evidence that survives process exit; a bare marker (Defect B) proves nothing. |
| worker plan node → sink_blocked persistence | A `sink_blocked` without an anchor would be a security-meaningless row; the TCB must refuse to persist it. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-21 | Repudiation / Tampering | `sink_blocked` persistence (server.rs) | high | mitigate | Anchor rides in the hashed `payload` column (`compute_event_hash` covers payload) → tamper-evident; `Event.taint == anchor.taint`, DB readers re-derive trust via `is_untrusted()` (no stored boolean). |
| T-07-22 | Spoofing (stapled taint) | executor block branch | high | mitigate | Anchor fields are exact clones of the resolved ValueRecord; the executor sets no taint (T-04-03). A stapled anchor would fail the 07-05 after-exit genuine-taint backstop. |
| T-07-23 | Tampering (non-persistable defect) | `append_event` | medium | mitigate | `append_event` rejects `sink_blocked` with `anchor == None` → Defect B is non-persistable through the TCB, not merely not-triggered. |

</threat_model>
