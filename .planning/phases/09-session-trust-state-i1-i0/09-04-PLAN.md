|----------|-------------|
| filesystem -> CLI | `--seed-from-file` reads untrusted external content as the intent seed — a new untrusted-input surface that must fail closed |
| CLI -> broker create_session | the CLI supplies provenance only; the broker (trusted path) sets the authoritative session status |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-09-10 | Spoofing / EoP | CLI (or its caller) self-declares session status instead of supplying provenance and letting the broker decide | high | mitigate | CLI passes only `SeedProvenance` to `create_session`; the broker computes `Draft`/`Active` (DESIGN §3); CLI forwards the broker's status, never sets it |
| T-09-11 | Input Validation (V5) | Malformed/missing seed file silently falls back to a trusted-arg (Active) session | high | mitigate | Fail-closed: missing/unreadable `--seed-from-file` path is a hard error, asserted by the integration test |
| T-09-SC | Tampering | Supply-chain (package installs) | low | accept | No `cargo add` / package installs this phase |
</threat_model>
