<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| planner → executor | The executor receives PlanArg handles and resolves authoritative records from the trusted store — it never trusts planner-supplied metadata. |
| executor → value store | Read-only at decision time; the store is mutated only by the mint path (the broker read-event handler), never by the executor. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Tampering | submit_plan_node / sink sensitivity map | critical | mitigate | Deterministic membership test: a non-empty-taint value in a routing-sensitive arg returns Block before any sink invocation (DESIGN-plan-executor §Executor Decision Logic). |
| T-04-03 | Spoofing | executor decision path | critical | mitigate | The executor only reads taint via resolve and never writes/staples a record; the Block payload is copied verbatim from the resolved record's provenance_chain. Asserted by the negative grep (no mint / no ValueRecord construction in lib.rs). |
| T-04-02 | Tampering | value store resolution | high | mitigate | A dangling/forged handle resolves to None and returns Denied — never Allowed — so an injected planner cannot reference a non-existent clean value to bypass. |
| T-04-SC | Tampering | cargo dependencies | low | accept | executor Cargo.toml uses only path + already-vetted workspace deps (uuid, serde, serde_json, anyhow). No new registry crate; no Package Legitimacy Gate required. |
</threat_model>
