<objective>
Build the real broker-resident SMTP adapter module `crates/brokerd/src/sinks/email_smtp.rs` and add the `lettre 0.11.22` dependency. This module is the ONLY code path that performs an SMTP call (D-03). It builds the wire message exclusively through lettre's typed `Message::builder()` setters (SMTP-05, D-07/D-22), sends via a no-TLS/no-auth local transport, and appends opaque-payload `email_send_succeeded` / `email_send_failed` events. This plan does NOT wire the adapter into `confirm()` — that is Plan 02 (which owns the atomic CAS + `email_send_attempted` transaction).

Purpose: Deliver a self-contained, unit-testable adapter with by-construction CRLF/header-injection defense before any confirm-path signature surgery.
Output: New adapter module, Cargo dependency, structural grep gate, and a parse-level CRLF-rejection unit test.
</objective>
