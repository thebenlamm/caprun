<objective>
Wire the `github.pr` Allowed-decision dispatch in `server.rs`, composing the
capability grant gate (GITHUB-02), the duplicate-PR CAS (GITHUB-04), and the sink
POST (GITHUB-01) into the never-blocked (untainted) path — mirroring the
email.send Allowed-dispatch arm's locking, head-advance, and
authorize-then-effect ordering.

bare Allowed decision is NOT enough — DESIGN §4.3/§8: absent grant → Deny), and a
replayed identical submission creates at most one PR via the content-derived CAS
committed BEFORE any socket opens (§4.5). Both gates sit ahead of the POST.

</objective>
