<objective>
Implement the `github.pr` sink adapter (Pattern A, broker-resident) and the POST
egress it needs, reusing the Phase-37 `http.request` SSRF resolve-and-pin
infrastructure verbatim. This plan writes the COMPLETE sink surface — the
Allowed-path invoker, the confirm-release invoker, and the confirm-release
precheck — so the two dispatch loci (Plans 38-04 server.rs, 38-05 confirmation.rs)
only wire calls and never edit this module.

Purpose (GITHUB-01): create a PR via one REST POST with a broker-held bearer
token, with the token in broker-local env only and OPAQUE audit payloads (DESIGN
§4.1/§4.2). The POST destination host is pinned (MAJOR-4): the API base is fixed
broker-owned trusted config, riding the SAME §3.6 resolve-and-pin as the GET path,
so an attacker-influenced owner/repo bounds only the URL path, never the host.
Output: a POST egress helper in http_request.rs + `sinks/github_pr.rs`.

Note: the live socket POST is Linux-gated with a macOS no-op stub, exactly like
`http_request::do_pinned_get` — live GitHub behavior is Phase 40 (mock endpoint).
github.pr does NOT mint (a created-PR response is not re-fetched as a GET), so no
new mint site is introduced and check-invariants Gate 3 is unchanged.
</objective>
