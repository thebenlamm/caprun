  <action>
    ARCHITECTURAL DECISION (planner, deviates from 03-PATTERNS.md §caprun pre_exec illustration): use the SELF-CONFINEMENT model. apply_confinement() is designed to be called by a process on ITSELF at startup, NOT inside Command::pre_exec before an exec. Rationale: Landlock deny-all also denies the Execute right and read of the target binary, and seccomp deny-execve blocks the launching execve itself — so applying either in pre_exec before exec would prevent the worker binary from ever launching. The worker process therefore confines itself after it has loaded and (optionally) connected to the broker. Record this rationale in the lib.rs top doc-block.

    rlimits.rs (`#[cfg(target_os="linux")]`): implement apply_rlimits() using `nix::sys::resource::{setrlimit, Resource}` — RLIMIT_AS = 512*1024*1024, RLIMIT_CPU = 30 (soft=hard). Map nix errors to std::io::Error per the pattern.

    landlock.rs (`#[cfg(target_os="linux")]`): implement deny_all_filesystem() per 03-RESEARCH.md Pattern 2 — `Ruleset::default().handle_access(AccessFs::from_all(ABI::V3)).create().restrict_self()` with NO allow rules (everything denied). Attempt ABI::V3; if the kernel reports less than fully enforced, fall back to ABI::V1 (Linux 5.13). eprintln the RulesetStatus. Map all errors to std::io::Error.

    seccomp.rs (`#[cfg(target_os="linux")]`): implement apply_worker_filter() using the EXACT seccompiler 0.5.0 API verified in Plan 01's doc-block — default action allow, deny (Errno EPERM) for execve, execveat, and socket when domain is AF_INET or AF_INET6 (use seccompiler argument-matching on the socket domain arg if the verified API supports it; otherwise deny socket broadly and document the tradeoff). Convert to BpfProgram and apply_filter.

    lib.rs: implement apply_confinement() for `#[cfg(target_os="linux")]` calling in MANDATORY order (03-RESEARCH.md Pitfall 2): (1) prctl(PR_SET_NO_NEW_PRIVS, 1) via nix, (2) rlimits::apply_rlimits(), (3) landlock::deny_all_filesystem(), (4) seccomp::apply_worker_filter(). Keep the `#[cfg(not(target_os="linux"))]` no-op. Keep the existing noop unit test.

    confine-probe.rs: a `[[bin]]` that reads argv[1] in {`fs`,`net`,`exec`}, calls `sandbox::apply_confinement()` on itself, then attempts the corresponding operation — fs: open `~/.ssh/id_rsa` (or `$HOME/.ssh/id_rsa`); net: `socket(AF_INET, SOCK_STREAM)` + connect to 1.1.1.1:80; exec: `execve("/bin/true")`. Exit code 0 IFF the operation was correctly BLOCKED (EACCES/EPERM); exit non-zero if the operation unexpectedly succeeded or errored differently. On non-Linux, exit with a sentinel code indicating "confinement not enforced (no-op)".

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| confined worker ↔ kernel | The worker is untrusted; the kernel (Landlock+seccomp) is the enforcement point |
| confined worker ↔ filesystem/network | Worker must have zero ambient access; only broker-passed fds are legitimate |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-03 | Tampering | Worker opens filesystem directly (bypass broker) | critical | mitigate | Landlock deny-all (no allow rules); `negative_fs` integration test proves open(~/.ssh) fails. |
| T-03-04 | Information Disclosure | Worker opens outbound network socket (exfiltration) | critical | mitigate | seccomp denies socket(AF_INET/AF_INET6); `negative_net` proves it. |
| T-03-05 | Elevation of Privilege | Worker execs a child to escape | critical | mitigate | seccomp denies execve/execveat; NO_NEW_PRIVS prevents privilege gain; `negative_exec` proves it. |
| T-03-06 | Spoofing/False-assurance | seccomp filter silently fails (EACCES) when NO_NEW_PRIVS not set | high | mitigate | Mandatory ordering: prctl(NO_NEW_PRIVS) before any seccomp call (Pitfall 2); apply_filter error is propagated, never swallowed. |
</threat_model>
