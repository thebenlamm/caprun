  <action>
    ARCHITECTURE (per Plan 02's self-confinement decision — Landlock deny-all + seccomp deny-execve cannot be applied in pre_exec before exec, because they would block loading/launching the worker binary). caprun therefore spawns `caprun-worker` as a NORMAL subprocess (no confinement in pre_exec); the worker confines ITSELF after connecting.

    worker.rs (compiles as the `caprun-worker` binary): read `BROKER_SOCK` and `SESSION_ID` from env; connect to the broker's abstract UDS socket; THEN call `sandbox::apply_confinement()` on itself (after this point: no fs, no net, no exec, bounded cpu/mem — but the already-open broker socket fd and any passed fds remain usable); send `BrokerRequest::RequestFd { path }` (the workspace file path, passed via env or argv) framed with the 4-byte LE length prefix; receive the fd out-of-band via `adapter_fs::recv_fd` on the socket's raw fd; read the file THROUGH the received fd (not via open — open is Landlock-denied); send `BrokerRequest::ReportRead { bytes_read }`; exit 0.

    main.rs (compiles as the `caprun` binary, `#[tokio::main]`): parse the workspace file path arg; open the audit DB (temp path or a CLI-provided path) via `brokerd::audit::open_audit_db`; create a Session via `brokerd::session::create_session` + persist it + append the `session_created` Event (parent_hash=None); start the broker UDS server bound to `\0/agentos/<session_id>`; spawn the `caprun-worker` subprocess (resolve its path; in tests use `env!("CARGO_BIN_EXE_caprun-worker")`) with `BROKER_SOCK`/`SESSION_ID`/workspace-path env vars. In the broker's connection handler, extend the Plan 03 dispatch: on `RequestFd { path }` -> the broker (which has ambient fs) opens the file, appends an `fd_granted` Event (parent = session_created hash), and passes the fd to the worker by running `adapter_fs::pass_fd(stream_raw_fd, file_fd)` INSIDE `tokio::task::spawn_blocking` (Pitfall 4 — sendmsg is blocking), then replies `FdGranted`; on `ReportRead { bytes_read }` -> append a `file_read` Event with taint = `[]` (clean workspace file; NOT stapled — Phase 4 adds genuine taint) parented to fd_granted, reply `Ack`. After the worker exits, print the audit DAG (events in causal order with their hashes) to stdout. Map all errors with anyhow; never unwrap in the server loop.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| confined worker ↔ broker | The worker's only egress; every effect is mediated and audited here |
| broker ↔ workspace fs | Only the broker opens files; the worker receives fds, never paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-14 | Repudiation | A worker read leaves no audit trail | high | mitigate | Every RequestFd/ReportRead appends an Event; dag_chain_integrity asserts the read Event is present and chained. |
| T-03-15 | Tampering | Worker bypasses the broker and opens the file directly | critical | mitigate | Worker self-confines (Landlock deny-all) BEFORE reading; it can only use the broker-passed fd — proven by substrate_demo. |
| T-03-16 | Spoofing | Demo "passes" via an LLM or hidden ambient access | high | mitigate | Demo is fully scripted/no-LLM; worker is confined; read flows through the passed fd; chain verified. |
| T-03-17 | DoS | Blocking sendmsg stalls the tokio broker | medium | mitigate | pass_fd runs in tokio::task::spawn_blocking (Pitfall 4). |
</threat_model>
