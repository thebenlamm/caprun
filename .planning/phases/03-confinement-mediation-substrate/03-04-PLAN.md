  <behavior>
    - pass_fd sends exactly one fd in a single ControlMessage::ScmRights with a 1-byte iov payload.
    - recv_fd receives the fd using a cmsg_space!([RawFd; 1]) buffer and sets FD_CLOEXEC on it before returning.
    - recv_fd returns ENODATA if no SCM_RIGHTS cmsg is present.
  </behavior>
  <action>
    Implement crates/adapter-fs/src/lib.rs per 03-RESEARCH.md Pattern 6 VERBATIM (this file has no codebase analog). `pass_fd(socket_raw_fd, file_raw_fd)`: build a single-byte IoSlice payload (some kernels require >=1 byte for cmsg delivery), put the single fd in one `ControlMessage::ScmRights(&[file_raw_fd])`, call `nix::sys::socket::sendmsg`. NEVER send multiple ScmRights messages in one sendmsg (nix issue #464). `recv_fd(socket_raw_fd)`: allocate the ancillary buffer with `nix::cmsg_space!([std::os::fd::RawFd; 1])`, call `recvmsg`, iterate `msg.cmsgs()`, and on `ControlMessageOwned::ScmRights(fds)` take the first fd, IMMEDIATELY `fcntl(fd, F_SETFD, FdFlag::FD_CLOEXEC)` (Pitfall 6), and return it; return `Err(Errno::ENODATA)` if no fd arrived. Implement src/protocol.rs `RequestFd { path: String, session_id: uuid::Uuid }` and `FdGranted { path: String }` with the standard derive stack. These are pure syscall wrappers — no `#[cfg(target_os="linux")]` gate needed (SCM_RIGHTS works on macOS too, which lets the round-trip test run on the dev machine), but document that in the real demo the worker is Linux-confined.
