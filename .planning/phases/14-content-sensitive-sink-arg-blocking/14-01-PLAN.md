<objective>
Extend I2 deterministic blocking from routing/recipient args to content args (email body), and do it via Collect-then-Block so a plan node with more than one sensitive+tainted arg surfaces ALL of them in one decision. This is the TCB type + logic layer of Phase 14.

Purpose: Close CONTENT-01 (the body-block gap deferred at v1.2) WITHOUT reintroducing the v1.2 B1 bug in a new form — a naive "flip Step 3 to Block" would let a plan node with a tainted `to` AND a tainted `body` block on `to` only, shipping the body unconfirmed. The fix (D-14) makes `ExecutorDecision::BlockedPendingConfirmation` / the anchor plural and scans every arg before returning.

Output: A plural, `Vec`-shaped block decision; a unified collect-then-Block loop; the `attachment` descope (D-23); and the executor-side proof tests. Downstream consumers (brokerd, event.rs, cli) are migrated in 14-02 (Wave 2).

Scope guard: This plan does NOT implement CONFIRM-03's combined-digest binding or CONFIRM-04's per-arg narration UX — those are Phase 16. This plan establishes only the plural DATA shape and the block behavior.
</objective>
