<objective>
Refine the executor's I2 blocking predicate (HARD-02): block only when a routing-sensitive arg's
resolved record carries an explicitly-untrusted taint label, so a UserTrusted/LocalWorkspace-only
record passes. This is the single change that makes the clean allow-path reachable at the executor.

Purpose: the current predicate `!record.taint.is_empty()` blocks ANY taint, including the positive
UserTrusted provenance the clean path mints. HARD-02 requires the predicate to be defined over
untrusted labels specifically.
Output: one-line predicate change + executor tests (UserTrusted allows; ExternalUntrusted still blocks).

## Artifacts this plan produces
- Executor blocking predicate rewritten over `TaintLabel::is_untrusted()` (from Plan 01)
- HARD-02 allow case + untrusted regression guard in executor_decision tests
</objective>
