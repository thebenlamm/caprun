<objective>
Wire the CLI end: parse a typed intent from the command line (PLAN-01), extract the deterministic
planner into `plan_from_intent` (PLAN-02) that holds only opaque ValueId handles (PLAN-03), and drive
the ProvideIntent round-trip from the confined worker so the clean path uses the UserTrusted handle.

Purpose: this closes the loop between user input and the broker-minted trusted value. The planner is a
pure mapping — no LLM, no raw bytes, no taint — keeping I0/I1 closed.
Output: planner.rs, updated main.rs arg parsing + INTENT env var, updated worker.rs protocol + planner call.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI/env-var → worker | INTENT env var carries the typed intent from trusted orchestrator to worker |
| worker → planner | Planner receives only the typed intent + opaque ValueId handles — never raw bytes or taint |
| confined worker → broker | ProvideIntent is sent after self-confinement; broker mints authoritatively |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-06 | Tampering | forged INTENT env var | medium | mitigate | Worker deserializes into an exhaustive enum (unknown variant → Err, fail closed); broker mints authoritatively; worker never constructs the ValueRecord |
| T-06-07 | Information Disclosure | planner sees raw bytes/taint | high | mitigate | plan_from_intent signature is (&CaprunIntent, ValueId, &[ValueId]) — type system forbids passing a ValueRecord/literal/taint (PLAN-03) |
| T-06-10 | Elevation of Privilege | IPC before confinement | medium | mitigate | Ordering invariant: apply_confinement() precedes ProvideIntent (Pitfall 6) |

No package installs in this phase; `T-06-SC` not applicable.
</threat_model>
