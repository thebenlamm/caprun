<objective>
Add the broker-owned clean-path minting: `mint_from_intent` (sibling of `mint_from_read`) that mints
a UserTrusted ValueRecord anchored to a new `intent_received` audit event, plus the `ProvideIntent` /
`IntentAccepted` IPC round-trip that drives it inside the per-connection ValueStore (PLAN-04).

executor never sees it (Pitfall 1 → silent Denied). The round-trip is the only correct path.
Output: mint_from_intent, two proto variants, one dispatch arm, and tests proving the genuine anchor.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker IPC → broker mint | Worker supplies intent literal via IPC; broker mints the ValueRecord authoritatively (worker never constructs it) |
| mint call → audit DAG | The mint appends the provenance-anchor event; a stapled/fabricated anchor would break the §9 genuineness proof |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-04 | Tampering | provenance anchoring | critical | mitigate | mint_from_intent appends the intent_received event AND mints in one call → provenance_chain[0] == that event id; test asserts identity (anti-stapling) |
| T-06-05 | Elevation of Privilege | per-connection ValueStore scope | high | mitigate | mint_from_intent called only from the ProvideIntent arm inside handle_connection; minting elsewhere → unresolvable handle → Denied (Pitfall 1) |
| T-06-09 | Tampering | taint laundering (same literal as a hostile record) | medium | accept | Each ValueId resolves independently; a UserTrusted record for a user-chosen recipient is legitimate and separate from any hostile record — humans choose their own recipients (RESEARCH Security Domain) |

No package installs in this phase; `T-06-SC` not applicable.
</threat_model>
