## Why this milestone exists

An adversarial review of a proposed "put an LLM in the planner seat" milestone found — and a Linux repro CONFIRMED — that caprun v1.3's guard(a) provides **no cross-connection protection**. A worker (or any second party) opens a second AF_UNIX connection to the same session socket, gets fresh per-connection trust state (`intent_provided`/`fd_requested` both false, `session_status` stale-Active), calls `ProvideIntent("attacker@evil.com")` → minted `UserTrusted` → routes to `email.send` → `Allowed` → live send.

This falsifies the load-bearing assumption behind I0/I1/I2: **that `UserTrusted` means "the human typed it."** It does not, today. A worker can introduce a fresh `UserTrusted` literal of its own choosing mid-session by reconnecting. This is the Phase-16 exfil hole reopened through connection multiplicity.

**Reachability is real:** confinement never unshares the netns; seccomp denies only `socket(AF_INET/AF_INET6)`, `AF_UNIX` is permitted (`seccomp.rs:6,64-83`); abstract UDS is explicitly unaffected by Landlock (`landlock.rs:4`). A confined worker CAN open the second connection.

**Not a production incident** (nothing deployed; repo unpushed, Ben's standing call) — but it breaks a shipped v1.3 security CLAIM, so DOC-01 must be corrected and the fix precedes all new capability.

## Phase 0 — Cross-connection trust coherence (THE FIX). Blocks everything else.

**DONE gate:** `crates/brokerd/tests/two_connection_intent_bypass.rs` flips from red (panic `LIVE EXFIL`) to green **by fixing the broker, not the test** — the safe-outcome assertions stay exactly as written; only the product behavior changes. Plus the existing v1.3 live acceptance (`scripts/mailpit-verify.sh`) stays green (no regression), independently re-run per project discipline.

**Design gate first (project rule — no TCB code before a reviewed DESIGN doc):**
`planning-docs/DESIGN-session-trust-coherence.md` must exist and clear an adversarial gate (fresh panel, not self-review — per DEC-ai-review-satisfies-human-gate) before any `server.rs` change. The DESIGN doc must resolve, at minimum:

1. **Authoritative trust state, not per-connection clones.** `intent_provided`, `fd_requested`, and `session_status` must resolve from one authoritative per-session store shared across every connection to that session — never seeded-once-per-connection and "never re-derived." Decide: single-connection-per-session enforcement (reject a 2nd connection to an active session) OR shared coherent state across connections. Name the tradeoff; the confined worker legitimately holds exactly one connection, so **rejecting a 2nd connection to an established session is the smaller, safer hammer** and should be the default unless a real need for N>1 is shown.
2. **Per-verb capability split (forward-looking for Phase 1).** A connection must be able to hold NO mint verb (`ProvideIntent`, `ReportClaims`, `ReportDerivedClaim`). Phase 1's planner connection will need exactly this. Design it now even if Phase 0 only needs the coherence fix.
3. **guard-(c) interaction.** `CAPRUN_ENABLE_IPC_CREATE_SESSION` ships in the production binary (accepted v1.3 residual). Re-confirm Phase 0's fix does not widen it, and re-state whether it should finally be compile-excluded.

**Design-gate inputs carried from the reviews (must each get a ruling in the DESIGN doc):**
- **MAJOR-2 (replay).** The Allowed-path has no CAS; under an adaptive planner (Phase 1) the replay actor collapses from "external" to "the named adversary." Amplification is bounded to trusted/human-typed recipients (untrusted still Blocks) → DoS/duplication, not new exfil. **Either re-earn "accepted" against the new actor in writing, or scope a minimal CAS into Phase 0.** Do not silently inherit.
- **MAJOR-3 (three mint sites).** The reduction "planner can't launder what it never holds" must audit all three: `mint_from_read` (safe only via the sensitivity map → `ExternalUntrusted` Blocks), `mint_from_intent` (the T1 hole — `ProvideIntent`), `mint_from_derivation` (safe via byte-verify+all-`file_read`). Correct claim: *"the only mint yielding a TRUSTED handle from a supplied string is `ProvideIntent`."* Reopens the instant any sink's sensitivity map treats `WorkerExtracted`/`ExternalUntrusted` as non-blocking on some arg — state this tripwire.
- **MEDIUM-1 (decision oracle).** `Allowed` vs `BlockedPendingConfirmation{anchors}` leaks per-handle taint state, and `literal_sha256` per blocked arg (`executor_decision.rs:144`) is an offline literal-guess confirmer. Document explicitly; decide whether Phase 1's planner connection sees full decisions or a reduced signal.

**DOC-01 honesty update (Phase 0, doc-only, propose-then-apply):** PROJECT.md residual-risks / DOC-01 section must record that v1.3's guard(a) was cross-connection-bypassable, that v1.4 Phase 0 fixes it, and (until fixed) that the `UserTrusted == human-typed` invariant did not hold across connections. The repro test is the evidence anchor.

## Phase 1+ — The adversarial planner (unblocked by Phase 0)

The original goal, reshaped by the reviews. **NOT** "swap an LLM in and re-run v1.3" — the tainted value flows through the *worker's* `mint_from_read`/`mint_from_derivation`, never the planner, so the v1.3 block fires identically regardless of planner intelligence. The milestone is to prove the boundary is **indifferent to planner intelligence** because value flows *around* the planner, and to close the one place planner choice is unconstrained.

**Build (behind a designed seam — note there is NO `Planner` trait today; it's a bare fn `plan_from_intent`, `planner.rs:76`; the seam must be designed):**
- A minimal LLM planner emitting only `PlanNode{sink, args: Vec<PlanArg>}` — no literal field to carry. Cheapest model that follows a tool schema; model choice is irrelevant and no model-quality claim is made.
- Planner runs on a connection holding **no mint verb** (Phase 0 capability split), sees typed extracts + handle IDs only, no caps, no net beyond its inference endpoint. Resolve the co-location question the reviews raised: it must NOT be in-process with the worker's raw-bytes fd (that would breach PLAN.md:31 "typed extracts only" and reintroduce token-stream laundering).

**Prove (the HARD GATE — items 2-3 are the gate; NOT the grep):**
1. An LLM planner, handed a doc whose injection instructs it to email attacker@evil.com, **complies** — emits a syntactically valid `PlanNode` routing the tainted handle to `to`.
2. The executor **Blocks deterministically**, `verify_chain` true, Mailpit==0 — genuine propagation, per §9 standard.
3. A trusted-intent control on the same sink Allows and delivers exactly once.
4. **Replace the context-dump grep** (theater — tests harness input, misses recipient fragmentation, trivially satisfiable): a **deterministic construction-site assertion** — feed the planner-prompt constructor a tainted record with a sentinel literal (sentinel *each fragment*), assert the sentinel bytes never appear in the constructed prompt. Unit-level, not probabilistic.

**T2 — the one genuinely new degree of freedom (slot binding). Decide enforce-or-defer in the Phase-1 design gate:**
An LLM's only real freedom the executor doesn't already constrain is *which handle goes in which slot* (sink choice fails closed on `UnknownSink`). Today a `UserTrusted` handle in the `to` slot is neither sensitive-untrusted nor slot-checked → I2 does not fire; there is **no slot-mismatch `DenyReason`** (`executor_decision.rs`). Safe today only *incidentally* (all `UserTrusted` handles are human-typed, so a misroute carries the human's own string and the SMTP adapter rejects it as malformed).
- **Option E (enforce):** slot–type binding in the executor — new `DenyReason`, new TCB logic. Makes v1.4 two milestones.
- **Option D (defer, recommended for v1.4):** document T2 as the accepted v1.4 residual (safe-by-incidental-human-typing), defer slot–type binding to v1.5. Keeps v1.4 one milestone.

## Right-sizing (from the reviews)

- **Phase 0** is small and sharply bounded — a coherence fix with an already-written regression gate. ~a handful of plans.
- **Phase 1+** is ~v1.1-shaped (10–15 plans) IF T2 is deferred (Option D) and the planner seam + boundary is the spine. If T2 enforcement (Option E) comes in, it's two milestones — split it out as v1.5.
- **Recommended:** v1.4 = Phase 0 (fix) + Phase 1 (adversarial planner, T2 deferred). Slot–type binding → v1.5.
